rv1126 + AW-NM372SM(海华模组)

  • 软件使用hostapd iw
  • 备注：
    • RTL模组： 使用p2p0作为softap功能，通过内核驱动的配置生成p2p0，如果没有p2p0节点请检查这里的配置。
    • AP/海华模组： 使用wlan1作为softap功能，且使用iw命令去生成wlan1节点。

0.启动网卡wlan0

killall connmand #如果有connmand需要先关闭
echo 1 > /sys/class/rfkill/rfkill1/state # 当蓝牙节点没有打开时是rfkill0

1.创建或修改节点为managed模式

  • 下面两种方式选择一种即可(方式一兼容性好,方式二可以同时使用热点和wifi)
1.修改原来的节点
iw dev wlan0 set type managed
  • 推荐使用这个方式
2.创建wlan1节点
  • 创建的时候需要wlan0是开启状态,否者会报错
ifconfig wlan0 up #需要先启用
wlan0 Link encap:Ethernet HWaddr F0:85:C1:0F:9C:02
UP BROADCAST MULTICAST MTU:1500 Metric:1
RX packets:0 errors:0 dropped:0 overruns:0 frame:0
TX packets:0 errors:0 dropped:0 overruns:0 carrier:0
collisions:0 txqueuelen:1000
RX bytes:0 (0.0 B) TX bytes:0 (0.0 B)
#创建wlan1
iw wlan0 interface add wlan1 type managed

2.启动hostapd

  • 先准备配置文件hostapd.conf参见下文
  • 启动程序hostapd -B /path/to/hostapd.conf #-B run daemon in the background
#无线设备名称(具体看用的上面哪种方式)
interface=wlan1

#使用的网桥名称，如果是用DHCP+NAT方式可忽略此行
#bridge=br0

#hostapd 0.6.8或者更高版本唯一选择
driver=nl80211

#终端看到的wifi名称，请自行修改
ssid=rv1126

#指明要选用的无线传输协议，这里表示使用802.11g
hw_mode=g

#802.11b/g都至多只有三条互不干扰的信道，即1,6,11，一般填这三个其中一个
channel=1

#验证身份的算法，1表示只支持wpa，2表示只支持wep,3表示两者都支持，wep已经被淘汰了，请不要使用。
auth_algs=1

#wpa加密方式，1代表支持wpa,2代表支持wap2,3代表两者都支持。
wpa=1

#wifi密码，请自行修改
wpa_passphrase=12345678

#对所有用户进行同样的认证，不进行单独的认证，如果需要，请设置为WPA-EAP。
wpa_key_mgmt=WPA-PSK

#控制支持加密数据的秘钥，CCMP比TKIP更强
wpa_pairwise=CCMP
